CONFIGURATION OF GITHUB

Basic Configuration Steps

  1. Install Git on your local machine.
  2. Create a GitHub account if you don't have one.
  3. Generate an SSH key pair for secure authentication.
  4. Add the SSH public key to your GitHub account settings.
  5. Clone a repository using SSH or HTTPS.
  6. Make changes and commit them locally.
  7. Push your changes to the remote repository on GitHub.

How to create a GitHub repository

  1. Log in to your GitHub account.
  2. Click the "New" button on the dashboard.
  3. Enter a name for your repository.
  4. Select whether it should be public or private.
  5. Initialize with a README file if desired.
  6. Create the repository.

How to configure first repository?

  1. Open your terminal or command prompt.
  2. Navigate to the directory where you want to create the repository.
  3. Run the command: git init
  4. Add files to the repository using: git add .
  5. Commit the files with a message: git commit -m "Initial commit"
  6. Link the local repository to the remote GitHub repository using: git remote add origin [repository URL]
  7. Push the changes to GitHub: git push -u origin main

Create a new repository on the command line

  1. Create a new directory for your project: mkdir [project-name]
  2. Navigate into the directory: cd [project-name]
  3. Initialize a new Git repository: git init
  4. Create a README file: echo "# [project-name]" >> README.md
  5. Add the README file to staging: git add README.md
  6. Commit the file: git commit -m "first commit"
  7. git branch -M main
  8. Link to the remote repository: git remote add origin [repository URL]
  9. Push the changes: git push -u origin main

Short cut commands:

  1. git init
  2. git add .
  3. git commit -m "Initial commit"
  4. git branch -M main
